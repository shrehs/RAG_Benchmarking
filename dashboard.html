Architecture Recommendations

HYBRID RAG - Maximum Recall
Best for accuracy-critical applications. Combines BM25 lexical search with semantic vector search. Recall: 1.80 | Latency: 188ms | Throughput: 5.4 q/s
VECTOR RAG - Recommended for Production
Best balance of recall, latency, and simplicity. Solid 1.50 recall with 75ms P50 latency. Use this as your default unless you have specific requirements. Recall: 1.50 | Latency: 75ms | Throughput: 13.0 q/s
GRAPH RAG - Ultra-Low Latency
Use when latency is critical and recall can be sacrificed. Pre-computed entity relationships enable lightning-fast retrieval. Warning: Entity resolution issues (Kubernetes) hurt recall. Recall: 0.80 | Latency: 9.4ms | Throughput: 127.8 q/s
PARENT-CHILD RAG - Large Documents
Useful for very large documents (>10K tokens). Embeds both parent context and child chunks. Massive indexing overhead (4.5+ minutes). Moderate recall improvement. Recall: 0.95 | Latency: 120ms | RAM: 3,663 MB
MULTI-QUERY RAG - Not Recommended
Makes 3 LLM calls per query for marginal recall improvement (1.50, same as Vector). 9× slower than Vector with no quality benefit. Skip unless specifically reducing query variance. Recall: 1.50 | Latency: 675ms | Throughput: 1.5 q/s

WHAT THE BENCHMARK MEASURED
─────────────────────────────────────────────────────────────────────

RETRIEVAL METRICS (Primary):
  - Recall@5: How many relevant docs found in top 5
  - Precision@5: % of top 5 that are relevant
  - MRR: Rank of first relevant document

SYSTEM METRICS:
  - P50 Latency: Median response time (milliseconds)
  - Throughput: Queries per second
  - Peak RAM: Memory usage during indexing
  - Storage: Index file size

QUALITY METRICS: SKIPPED
  - Faithfulness: Dropped (unreliable with 8B models)
  - Answer Relevancy: Only with --ragas flag
  - Reason: Wants to compare architectures, not LLM quality

IMPORTANT LIMITATIONS
─────────────────────────────────────────────────────────────────────

1. Synthetic QA Pairs
   - RAGAS-generated questions != real user queries
   - Results may differ with actual human queries

2. Faithfulness Metric Missing
   - Intentionally dropped (unreliable with 8B models)
   - Use GPT-4o if hallucination detection needed

3. Entity Resolution Issues (Graph RAG)
   - Kubernetes "hub node" (90%+ docs contain it)
   - May perform better on other domains

4. Single LLM Model
   - Only tested Groq llama-3.1-8b
   - GPT-4o/Claude-3 may show different patterns

5. Small Dataset
   - 50 Wikipedia articles (may not scale)
   - Trends likely valid, but needs validation
